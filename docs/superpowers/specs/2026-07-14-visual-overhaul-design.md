# 비주얼 개편 설계 문서 (코믹 오피스 테마 + 폰트 확대)

## 배경

"그래픽이 허접하다"는 피드백 반영. 원인: (1) 배경/패널이 전부 무채색 회색
하나뿐이고 UI 요소(버튼/바/패널)에 스타일이 전혀 없음, (2) 캐릭터가 이모지
(유령 이모지가 김대리) 뿐이라 통일감 없음, (3) 글자가 전반적으로 작음(프로젝트
기본 폰트 크기 16px, 일부는 12px로 더 작게 오버라이드됨).

사용자가 선택한 방향: **코믹 오피스** — 밝고 채도 높은 코미디톤.

## 1. 색상 팔레트 (GameData.gd 또는 별도 Palette 상수)

- 배경: 크림/라이트 옐로우 그라디언트 `#FFF6E5` → `#FFE9C2`
- 카드/패널 배경: `#FFFFFF` (살짝 반투명 가능)
- 텍스트: 진한 남색/차콜 `#2B2440` (검정 대신 — 코믹 톤에 더 잘 어울림)
- 포인트 컬러 3종:
  - 오렌지(주 액션/버튼) `#FF6B35`
  - 민트(보조 액션/성공) `#2EC4B6`
  - 옐로우(강조/골드) `#FFD23F`
- 위험/적 HP `#E63946`, 플레이어 HP `#06D6A0`
- 패널 테두리: 진한 남색 `#2B2440` 2~3px (코믹북 아웃라인 느낌)

## 2. 타이포그래피

- Pretendard-Bold 추가 다운로드(제목/버튼/숫자용). 본문은 기존 Regular 유지.
- 전역 기본 폰트 크기를 `gui/theme/custom_font_size` 프로젝트 설정으로
  22px로 상향(기존 16px). 스테이지 타이틀/재화 숫자는 Bold 26~28px, 보스
  대사/로그는 18px, 리스트 행(장비/결사대 등)은 20px 이상으로 통일.
- 12px로 낮춰뒀던 부가 텍스트(뽑기 천장, 스킬 쿨다운 등)도 최소 16px로.

## 3. 패널/버튼/바 스타일 (Theme 리소스, 코드로 구성)

- 모든 패널: `StyleBoxFlat` 흰 배경 + 남색 테두리 2px + 모서리 radius 16px +
  은은한 그림자(shadow_size)
- 버튼: 기본은 오렌지/민트 배경 + 흰 텍스트 + radius, hover는 밝게, pressed는
  살짝 아래로 눌리는 오프셋(그림자 줄임)로 "꾹 눌림" 느낌
- ProgressBar: track은 연한 회색, fill은 상황별 컬러(적=빨강, 플레이어=초록,
  스킬 쿨다운=오렌지), radius 적용
- 기존 `EquipmentPanel`/`TeamPanel`/`GachaPanel`/`PrestigePanel`/
  `AchievementPanel`은 큰 폰트에 맞춰 행 높이/최소너비 재조정, `GachaPanel`·
  `PrestigePanel`도 `ScrollContainer`로 감싸 오버플로 방지

## 4. 커스텀 캐릭터 아트 (SVG, `assets/art/`)

플랫 벡터 스타일로 직접 그려서 `TextureRect`로 교체 (기존 이모지 Label
대체). 스타일 통일: 둥근 머리 + 심플한 몸통, 코믹한 표정, 각 캐릭터별
포인트 컬러 의상.

- `player.svg` — 김대리: 하늘색 셔츠+넥타이, 땀방울 하나(고생 포인트)
- `boss_1.svg` — 낙하산 박낙하 대리: 슬릭 헤어 + 선글라스 + 금색 낙하산 배지
- `boss_2.svg` — 라떼는 과장: 콧수염 + 커피잔 들고 있는 아저씨
- `boss_3.svg` — 트렌디 최부장: 힙한 선글라스 + 후드/캐주얼 정장

잡몹(9종)·재화 아이콘(💰😤💳🔒 등)은 이모지 유지 — 비중이 작아 공수 대비
효과가 낮음.

## 5. 레이아웃 조정

- `BattleView.custom_minimum_size` 360 → 420 (캐릭터 아트 + 커진 폰트 여유)
- `enemy_box`/`player_box` 아이콘을 `Label`(48px 이모지) → `TextureRect`
  (약 96~110px 정사각) 로 교체
- `panel_host` 높이 200 → 260 (큰 폰트로 늘어난 행 높이 수용)
- `GachaPanel`, `PrestigePanel`에 `ScrollContainer` 추가

## 검증

기존과 동일하게: 코드 작성 후 `godot --headless --editor --quit`으로 파싱
에러 확인 → export → Playwright로 전체 탭 스크린샷 재확인(레이아웃 깨짐/
글자 잘림 여부 특히 주의).
