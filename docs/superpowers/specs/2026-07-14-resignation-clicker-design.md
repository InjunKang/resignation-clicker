# 사직서 던지기! 대리님 키우기 — 설계 문서 (Phase 1 MVP)

## 배경

대한민국 직장인 대리 만족 컨셉의 방치형(idle/incremental) 클리커 게임. 사용자가 제공한
전체 기획서(스테이지/보스, 장비, 결사대, 액티브 스킬, 재테크 미니게임, 뽑기, 사직서
프레스티지)는 매우 방대하므로, 실제로 플레이 가능한 코어 루프를 갖춘 Phase 1 MVP를
먼저 구현하고 나머지는 로드맵으로 문서화한다.

## 기술 스택

- **엔진**: Godot 4.7 (GDScript, C# 아님)
- **타깃**: 웹(HTML5) export — Godot 웹 export는 `user://`가 브라우저 IndexedDB에
  매핑되어 저장이 영속됨
- **아트**: 별도 에셋 없이 ColorRect/Label/이모지 텍스트 기반 플레이스홀더 비주얼로
  구현. 나중에 스프라이트로 교체 가능하도록 시각 요소는 `TextureRect`/`ColorRect`
  자리만 잡아둠
- **구조**: Autoload 싱글톤으로 데이터/상태/저장을 분리 (`GameData`, `GameState`,
  `SaveManager`)

## 범위 (Phase 1에 포함)

1. 3분할 UI: 상단 재화바 / 중앙 전투 씬 / 하단 탭 메뉴
2. 능력치 4종: 타자속도(공격력) / 멘탈(체력) / 아부력(방어력) / 월급루팡(치명타%)
3. Stage 1-1~1-10: 일반 몹 3종 자동 처리 → 10스테이지마다 보스(낙하산 박낙하 대리)
   제한시간 전투
4. 장비 4종(키보드/의자/마우스/드링크) × 4티어, 골드로 레벨업
5. 재화: 월급(Gold, 일반) / 스트레스(Soul, 특수) — 법인카드(Diamond)는 자리만 만들고
   실제 결제(IAP)는 미구현
6. 월급루팡 탭 미니게임: 화면 연타 시 버스트 보너스(스트레스 획득)
7. 오프라인 방치 보상: 마지막 저장 시각 대비 경과 시간 기반 골드 정산(상한 캡 적용)
8. 저장/불러오기: `user://save.json`
9. 잠금 탭 4개(결사대/재테크/뽑기/사직서)는 "준비 중" 표시만 — 실제 기능 없음을
   명확히 표시(가짜 기능 없음)

## 범위 밖 (Phase 2+ 로드맵, 이번엔 미구현)

- Stage 2 (꼰대 플래닛) / Stage 3 (네카라쿠배당토) 및 각 보스
- 사내 비밀 결사대(동료 5종, DPS 지원 + 패시브)
- 액티브 스킬 4종(넵! 알겠습니다 / 커피 수혈 / 네 탓이오 / 전무님 라인 타기)
- 실시간 재테크 미니게임(가짜 주식/코인 시세)
- 장비 뽑기(가챠) 시스템, 법인카드(다이아) 결제
- 사직서 던지기(프레스티지/환생) 및 영구 버프 특성 상점

## 아키텍처

```
res://
  project.godot
  scenes/
    Main.tscn            # 루트: TopBar + Battle + TabMenu 배치
    Battle.tscn           # 전투 씬 (몹/보스 vs 김대리)
    ui/
      TopBar.tscn
      TabMenu.tscn
      panels/
        StatsPanel.tscn
        EquipmentPanel.tscn
        LockedPanel.tscn  # 결사대/재테크/뽑기/사직서 공용 잠금 표시
  scripts/
    autoload/
      GameData.gd         # 정적 데이터: 스테이지/몹/보스/장비 테이블
      GameState.gd         # 런타임 상태: 재화, 능력치, 장비 레벨, 스테이지 진행
      SaveManager.gd       # 저장/불러오기, 오프라인 보상 계산
    Main.gd
    Battle.gd
    ui/top_bar.gd
    ui/tab_menu.gd
    ui/panels/stats_panel.gd
    ui/panels/equipment_panel.gd
```

### 데이터 흐름

- `GameData`는 순수 정적 테이블(Dictionary/Array 상수)만 가지며 상태를 갖지 않는다.
- `GameState`는 `GameData`를 참조해 현재 능력치(장비 보너스 합산 포함)를 계산하고,
  변경 시 Signal(`currency_changed`, `stats_changed`, `equipment_changed`,
  `stage_changed`)을 emit한다. UI는 이 Signal을 구독해 갱신한다(폴링 없음).
- `Battle.gd`는 `_process(delta)`에서 공격 타이머를 누적해 `GameState`의 공격력으로
  현재 적 HP를 깎고, 적 사망 시 `GameState.add_gold()` 호출 후 다음 적을 스폰한다.
  보스는 제한시간 타이머를 별도로 가지며, 시간 초과 시 패배 처리(멘탈 데미지 후
  재시도) 로직을 갖는다.
- `SaveManager`는 앱 시작 시 `user://save.json`을 읽어 `GameState`에 복원하고,
  `last_save_unix`와 현재 시각의 차를 오프라인 골드 생산량(초당 생산량 × 경과초,
  상한 캡)으로 환산해 지급한다. 저장은 일정 주기(예: 30초)와 앱 종료 시 트리거.

### 전투 루프 상세

- 공격 주기(초) = `1.0 / attack_speed`. `attack_speed`는 타자속도 스탯에서 유도.
- 일반 몹: HP를 소진시키면 즉시 처치, 골드 보상 후 스테이지 내 다음 몹 스폰. 10마리
  처치 시 보스 등장.
- 보스: 제한시간 내에 HP를 0으로 만들어야 클리어. 실패 시 멘탈(플레이어 HP) 데미지를
  입고 몹 페이즈부터 재시도. 클리어 시 다음 스테이지로 진행하고 골드/스트레스 보상.
- 월급루팡(치명타): 매 공격마다 확률적으로 5배 데미지 + 스트레스 보너스.
- 탭 미니게임: 전투 화면 탭 시 즉시 소량의 스트레스 획득(쿨다운 없는 가벼운 보너스,
  연타를 유도하되 방치형 밸런스를 깨지 않도록 골드에는 영향 없음).

### 저장 데이터 스키마 (JSON)

```json
{
  "gold": 0,
  "stress": 0,
  "diamond": 0,
  "stage_index": 0,
  "equipment_levels": { "keyboard": 0, "chair": 0, "mouse": 0, "drink": 0 },
  "last_save_unix": 0
}
```

## 에러 처리 / 엣지 케이스

- 저장 파일이 없거나 손상된 경우: 기본값(신입사원 상태)으로 시작.
- 오프라인 경과 시간이 비정상적으로 큰 경우(시스템 시계 조작 등): 오프라인 보상은
  최대 8시간 분량으로 캡.
- 장비 레벨이 최대 티어에 도달하면 업그레이드 버튼 비활성화.
- 보스 제한시간 내 미처치: 재시도 가능(무한 재시도 허용, 페널티는 시간 손실만).

## 테스트/검증 계획

- Godot는 CLI(`--headless --check-only` 유사 방식은 없으므로 `--headless -e --quit`
  또는 커스텀 GDScript 유닛 테스트 스크립트)로 스크립트 파싱 오류를 확인.
- 설치된 Export Templates로 HTML5 export를 headless 커맨드로 실행해 실제 export
  결과물이 생성되는지 확인.
- GUI 에디터 조작(마우스 클릭 등)은 에이전트가 수행할 수 없으므로, 씬 트리 구성은
  전부 `.tscn` 텍스트 포맷을 직접 작성한다. export 완료 후 실제 브라우저 플레이
  테스트는 사용자가 직접 확인 필요 — 이 부분은 명시적으로 안내한다.
