# Phase 2 설계 addendum

Phase 1 spec(`2026-07-14-resignation-clicker-design.md`)의 "범위 밖(로드맵)"
항목을 구현하기 위한 결정사항. 원본 기획서가 이미 매우 상세하므로, 여기서는
구현 과정에서 필요한 구체적 판단(스케일링/수치 매핑)만 기록한다.

## 1. Stage 2·3 확장

Phase 1은 스테이지를 "회사 1개 × 10서브스테이지"로 하드코딩했다. 이를
`GameData.COMPANIES` 배열(3개 회사)로 일반화한다.

- `company_index = stage_index / STAGES_PER_COMPANY (10)`, 3개 회사 이후에는
  `min(company_index, 2)`로 클램프해 3번째 회사(네카라쿠배당토) 비주얼로
  무한 확장(성장 곡선은 stage_index 기반이라 계속 우상향). "게임 클리어"
  화면은 만들지 않음 — 방치형 관례상 무한 스케일링 + 프레스티지로 대체.
- 각 회사: `name`, `mobs[3]`, `boss{name, icon, quote, time_limit}`.

## 2. 사내 비밀 결사대 (동료)

원본 기획서의 해금 조건("1-5", "1-20", "2-10", "3-1", "4-5")은 회사당
10스테이지 구조와 맞지 않으므로, 0-based 글로벌 `stage_index`(0~29+)로
재매핑한다: 인턴 idx≥4, 복사기 기사 idx≥9, 동기 idx≥10, 이모님 idx≥20,
내부고발자 idx≥24. 해금은 자동(별도 재화 소모 없음) — 스테이지 도달 시
GameState가 즉시 unlock.

- 동료 DPS는 전투 tick마다 플레이어 데미지에 가산(합연산).
- 패시브는 각각 다른 계산식에 개입: 인턴(공격력 %), 복사기 기사(장비 강화
  비용 %), 동기(오프라인 수익 %), 이모님(보스 제한시간 +초), 내부고발자
  (치명타 배율 %).

## 3. 액티브 스킬

**단순화 결정**: 수동 탭 활성화 UI는 Phase 3로 미루고, 4종 모두 "오토"
모드만 구현한다(쿨타임 되는 즉시 자동 발동). 전투 화면에 4개의 쿨다운
표시만 추가. 스킬 해금은 스테이지 임계값으로 순차 해금(0/9/14/19).

## 4. 재테크 미니게임

가짜 종목 2개("삼송전자", "비트코인족")를 하드코딩. 실시간 가격은 3초마다
±random walk, 낮은 확률로 "상장폐지"(가격 0, 투자금 손실) 이벤트. 골드
일부를 투자/회수하는 매수·매도 버튼만 제공(차트 시각화는 텍스트 가격
추이 로그로 대체 — 그래프 렌더링은 Phase 3).

## 5. 뽑기 (가챠)

법인카드(다이아) 최초 획득처를 추가해야 함 — 각 회사 보스 최초 처치 시
5개 지급. 가챠는 1회 = 다이아 10개 소모 = 무작위 장비 슬롯 1개의 레벨을
+1~+3 랜덤 상승. 등급 연출/천장 시스템은 Phase 3.

## 6. 사직서 던지기 (프레스티지)

`stage_index >= 20`(3번째 회사 진입)부터 버튼 활성화. 실행 시 gold/
stress/diamond/stage_index/equipment_levels/companions/skill 쿨다운을
초기화하고, `floor(max_stage_reached / 2)`개의 "황금 사직서"(영구 재화)를
지급. 영구 특성 상점은 2개만 구현: 기본 공격력 +10%/스택(영구), 오프라인
수익 +10%/스택(영구) — 나머지 특성은 Phase 3.

## 검증

각 서브시스템 추가 후 `godot --headless --editor --quit`으로 스크립트
파싱 에러를 확인하고, 전체 구현 완료 후 Playwright 헤드리스 브라우저로
스크린샷 + 콘솔 에러 재검증한다.
